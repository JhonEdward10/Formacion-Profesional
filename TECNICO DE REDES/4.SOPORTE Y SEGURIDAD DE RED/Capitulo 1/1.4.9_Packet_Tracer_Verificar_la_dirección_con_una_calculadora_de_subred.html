Incompleto ​Packet Tracer - Verificar la dirección con una calculadora de subred

Objetivos

Las calculadoras de subredes pueden ser útiles para proporcionar información sobre una red o subred específica. Por lo general, una calculadora de subred solicitará una dirección IPv4 o IPv6 específica junto con su máscara de subred o la longitud del prefijo. Luego calculará los rangos de direcciones para el rango completo de subredes especificadas.

En esta actividad, determinará la dirección IPv4 y la máscara de subred de su dispositivo y utilizará una calculadora de subred en línea para determinar la dirección de red IPv4.

Instrucciones

Parte 1: Determinar la dirección IPv4 y la máscara de subred de su dispositivo

Dependiendo del sistema operativo de su dispositivo (computadora o teléfono móvil), use el comando adecuado para determinar lo siguiente:

  • Dirección IPv4
  • Máscara de subred

Registre la dirección IPv4 y la máscara de subred para su dispositivo.

Parte 2: Buscar una calculadora de subred en línea

  1. Usando su navegador web, busque la calculadora de subred IP para obtener una lista de varias calculadoras de subred IP.
  2. Elija una calculadora de subred IP de los resultados de búsqueda web.

Parte 3: Ingresar la dirección IPv4 y la máscara de subred

En el área correspondiente, ingrese la dirección IPv4 y la máscara de subred de su dispositivo. Realice el cálculo para determinar la dirección de red IPv4.

Parte 4: Examinar la dirección de red IPv4

La información real proporcionada variará, pero la información más común incluye:

  • Dirección de red
  • Rango de direcciones de host
  • Dirección de broadcast
  • Número de hosts utilizables

La dirección de red IPv4 que se muestra es la dirección de red de su dispositivo. Si la salida incluye un rango de direcciones de host, la dirección de su dispositivo debe estar dentro de ese rango.

Esta información puede ser útil para verificar:

  • Si el dispositivo en cuestión está en la misma red que su puerta de enlace predeterminada.
  • Si dos dispositivos están en redes diferentes y, por lo tanto, se debe utilizar el enrutamiento adecuado para la conectividad.

    • Nota: Muchas de estas calculadoras de subred también darán la clase de dirección; Clase A, B o C. Es importante señalar que esta información ya no es relevante. El direccionamiento con clase se suspendió con la introducción del enrutamiento entre dominios sin clase (CIDR) en 1993.
